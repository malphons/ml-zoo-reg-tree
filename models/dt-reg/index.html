Decision Tree Regressor

Regression Tree Est. 1984

Recursively partitions the feature space into regions and predicts the mean value within each region. The piecewise-constant step function shows how deeper trees capture finer nonlinear patterns.

max_depth: 2

Overview

A Decision Tree Regressor recursively splits the input space into rectangular regions. At each internal node, the algorithm selects the feature and threshold that minimizes the mean squared error of the resulting child nodes. Leaf nodes predict the average target value of all training points that fall into that region.

When to Use

  • You need an interpretable, rule-based model
  • The data has nonlinear relationships and interactions
  • Feature importance is needed for analysis
  • You want a quick baseline before trying ensembles

Key Characteristics

  • Non-parametric — makes no assumption about functional form
  • Greedy — locally optimal splits at each node
  • Piecewise constant — predictions are step functions
  • Depth controls complexity — deeper trees fit closer but risk overfitting

Pros

  • Highly interpretable — visualize as flowchart
  • Handles nonlinear relationships naturally
  • No feature scaling required
  • Automatic feature selection via split importance
  • Handles missing values (in some implementations)

Cons

  • High variance — small data changes alter the tree
  • Prone to overfitting without depth limits
  • Cannot extrapolate beyond training range
  • Predictions are discontinuous step functions
  • Greedy splits are not globally optimal

How It Works

The CART algorithm for regression builds a binary tree top-down:

  1. Start with all data points in the root node.
  2. Find the best split: For each feature and each possible threshold, compute the weighted MSE of the two child groups. Choose the split that gives the largest MSE reduction.
  3. Recurse: Apply the same procedure to each child node.
  4. Stop when a stopping criterion is met (max depth, min samples per leaf, etc.).
  5. Predict: For a new point, traverse the tree and return the mean target value of the leaf it lands in.

Use the max_depth slider above to see how increasing depth creates finer partitions. At depth 1 the tree makes a single split; at depth 4 it captures much of the underlying nonlinear pattern but may start overfitting.

Key Equations

Prediction in region R_m

$$\hat{y}_{R_m} = \frac{1}{|R_m|} \sum_{x_i \in R_m} y_i$$

MSE Criterion (Impurity)

$$\text{MSE}(R) = \frac{1}{|R|}\sum_{x_i \in R} (y_i - \hat{y}_R)^2$$

Variance Reduction at Split

$$\Delta = \text{MSE}(R) - \frac{|R_L|}{|R|}\text{MSE}(R_L) - \frac{|R_R|}{|R|}\text{MSE}(R_R)$$

Best Split Selection

$$(j^*, t^*) = \arg\max_{j, t} \;\Delta(j, t)$$

where $j$ is the feature index and $t$ is the threshold value.

Overall Prediction

$$\hat{f}(x) = \sum_{m=1}^{M} \hat{y}_{R_m} \cdot \mathbf{1}(x \in R_m)$$

scikit-learn

Python
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, r2_score

# Split data
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Fit model
model = DecisionTreeRegressor(
    max_depth=4,
    min_samples_split=5,
    min_samples_leaf=2,
    random_state=42
)
model.fit(X_train, y_train)

# Predict
y_pred = model.predict(X_test)

# Evaluate
print(f"R-squared: {r2_score(y_test, y_pred):.4f}")
print(f"RMSE:      {mean_squared_error(y_test, y_pred, squared=False):.4f}")

# Feature importance
for name, imp in zip(feature_names, model.feature_importances_):
    print(f"  {name}: {imp:.4f}")

Visualize the Tree

Python
from sklearn.tree import export_text, plot_tree
import matplotlib.pyplot as plt

# Text representation
print(export_text(model, feature_names=feature_names))

# Plot
fig, ax = plt.subplots(figsize=(16, 8))
plot_tree(model, feature_names=feature_names, filled=True,
          rounded=True, ax=ax, fontsize=8)
plt.tight_layout()
plt.show()
