LightGBM Regressor

Fast gradient boosting using histogram-based splits and leaf-wise tree growth. Optimized for efficiency on large datasets with high cardinality features.

Boosting rounds: 3
Learning rate: 0.3

Overview

LightGBM (Light Gradient Boosting Machine) is Microsoft's high-performance gradient boosting framework. It uses two key innovations: histogram-based splitting (bins continuous features into discrete buckets for faster split finding) and leaf-wise tree growth (grows the leaf with the largest loss reduction rather than level-by-level). These make it significantly faster than traditional implementations while achieving comparable or better accuracy.

When to Use

  • Large datasets where XGBoost is too slow
  • High-cardinality categorical features
  • You need fast training without sacrificing accuracy
  • Tabular/structured data regression tasks

Pros

  • Fastest gradient boosting implementation
  • Memory-efficient histogram representation
  • Native categorical feature support
  • Handles large datasets with millions of rows
  • GOSS reduces computation with minimal accuracy loss

Cons

  • Leaf-wise growth can overfit on small datasets
  • More hyperparameters to tune than simpler models
  • Less interpretable than a single tree
  • Histogram binning loses some precision
  • Sensitive to num_leaves parameter

How It Works

Histogram-Based Splitting

Instead of evaluating every unique value as a potential split point (as XGBoost does), LightGBM bins continuous features into a fixed number of discrete buckets (default 255). Split finding then operates on the histogram, reducing time complexity from O(n·features) to O(bins·features).

Leaf-Wise (Best-First) Growth

Traditional gradient boosting grows trees level-by-level. LightGBM instead always splits the leaf with the highest loss reduction. This produces asymmetric trees that can be deeper on one side, capturing complex patterns with fewer total leaves.

GOSS & EFB

Gradient-based One-Side Sampling (GOSS): Keeps all samples with large gradients but randomly samples from those with small gradients, since small-gradient samples are already well-learned.

Exclusive Feature Bundling (EFB): Bundles mutually exclusive sparse features together, reducing the effective number of features.

Key Equations

Gradient Boosting Objective

$$\mathcal{L}^{(t)} = \sum_{i=1}^{n} l(y_i, \hat{y}_i^{(t-1)} + f_t(x_i)) + \Omega(f_t)$$

Histogram Split Gain

$$\text{Gain} = \frac{1}{2}\left[\frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L+G_R)^2}{H_L+H_R+\lambda}\right] - \gamma$$

where $G$ and $H$ are sums of first and second order gradients in each bin.

Leaf-Wise Growth Rule

At each step, split the leaf $l^*$ where: $$l^* = \arg\max_{l \in \text{leaves}} \max_{j,t} \text{Gain}(l, j, t)$$

Additive Model

$$\hat{y}^{(T)} = \hat{y}^{(0)} + \eta \sum_{t=1}^{T} f_t(x)$$

where $\eta$ is the learning rate (shrinkage) and $f_t$ is the tree at round $t$.

LightGBM

Python
import lightgbm as lgb
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, r2_score

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

model = lgb.LGBMRegressor(
    n_estimators=100,
    learning_rate=0.1,
    num_leaves=31,
    max_depth=-1,
    min_child_samples=20,
    subsample=0.8,
    colsample_bytree=0.8,
    reg_alpha=0.1,
    reg_lambda=0.1,
    random_state=42,
    n_jobs=-1
)
model.fit(X_train, y_train,
          eval_set=[(X_test, y_test)],
          callbacks=[lgb.early_stopping(10)])

y_pred = model.predict(X_test)
print(f"R-squared: {r2_score(y_test, y_pred):.4f}")
print(f"RMSE:      {mean_squared_error(y_test, y_pred, squared=False):.4f}")
