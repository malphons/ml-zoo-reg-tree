Extra Trees Regressor

Regression Tree Est. 2006

Like Random Forest but with fully random split thresholds, reducing variance at the cost of slightly higher bias. Toggle between random and optimized splits to see the difference.

Overview

Extremely Randomized Trees (Extra Trees) pushes the randomization idea of Random Forest further. Instead of searching for the best split threshold at each node, it draws thresholds completely at random for each candidate feature, then picks the one that gives the best criterion value. This additional randomness further reduces variance and is computationally cheaper since no exhaustive threshold search is needed.

When to Use

  • You want faster training than Random Forest
  • The data is noisy and you want extra variance reduction
  • High-dimensional feature spaces
  • When slight bias increase is acceptable for lower variance

Pros

  • Faster training than RF (no exhaustive split search)
  • Lower variance than RF due to extra randomness
  • Handles noisy data well
  • Uses full training set (no bootstrap by default)
  • Parallelizable like RF

Cons

  • Slightly higher bias than RF
  • May need more trees to converge
  • No OOB estimate (uses all data, no bootstrap)
  • Harder to interpret individual trees
  • Random splits can be suboptimal on small datasets

How It Works

The key difference from Random Forest is in step 2:

  1. No bootstrap: By default, Extra Trees uses the entire training set for each tree (though bootstrap can be enabled).
  2. Random thresholds: At each node, select $K$ random features. For each feature, draw a single random threshold uniformly between the feature's min and max in that node. Pick the (feature, threshold) pair with the best criterion.
  3. Aggregate: Average the predictions from all trees, just like RF.

Click the buttons above to compare: Random Splits shows a single ET-style tree with randomly chosen thresholds, Best Splits shows the RF-style tree with optimized thresholds, and ET Ensemble shows 5 random-split trees averaged together.

Key Equations

Random Threshold Selection

For feature $j$ at node $t$, draw threshold:

$$t_j \sim \text{Uniform}\!\left(\min_{x_i \in R_t} x_{ij},\; \max_{x_i \in R_t} x_{ij}\right)$$

Split Selection

Among $K$ random (feature, threshold) pairs, choose:

$$(j^*, t^*) = \arg\max_{j \in S_K} \Delta(j, t_j)$$

where $S_K$ is the random feature subset of size $K$.

Ensemble Prediction

$$\hat{f}_{ET}(x) = \frac{1}{B}\sum_{b=1}^{B}T_b(x)$$

Bias-Variance Trade-off

Compared to RF, Extra Trees has:

$$\text{Bias}_{ET} \geq \text{Bias}_{RF}, \quad \text{Var}_{ET} \leq \text{Var}_{RF}$$

The increased randomness reduces correlation between trees, lowering ensemble variance.

scikit-learn

Python
from sklearn.ensemble import ExtraTreesRegressor
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, r2_score

# Split data
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Fit model
model = ExtraTreesRegressor(
    n_estimators=100,
    max_depth=None,
    min_samples_split=5,
    max_features='sqrt',
    bootstrap=False,   # default: use full dataset
    n_jobs=-1,
    random_state=42
)
model.fit(X_train, y_train)

# Predict
y_pred = model.predict(X_test)

# Evaluate
print(f"R-squared: {r2_score(y_test, y_pred):.4f}")
print(f"RMSE:      {mean_squared_error(y_test, y_pred, squared=False):.4f}")

# Feature importance
for name, imp in sorted(zip(feature_names, model.feature_importances_),
                         key=lambda x: -x[1]):
    print(f"  {name}: {imp:.4f}")
